class py_gjapi.GameJoltTrophy(username, user_token, game_id, private_key)[source]¶

The Class constructors. The class requires four essential parameters: user name, user token, game ID and private code. Check the API documentation on Game Jolt’s website to see what those parameters they are. In this code, I used the same names on the site. If you read it, you can understand what’s going on here.

Note that username and user token can be changed later, but the game id and the private key must be defined first, as they won’t change.

openSession()[source]¶

Opens a game session for a particular user. Allows you to tell Game Jolt that a user is playing your game. You must ping the session (pingSession method) to keep it active and you must close it when you’re done with it. Note that you can only have one open session at a time. If you try to open a new session while one is running, the system will close out your current session before opening a new one. Return a boolean value: True if a session opened with sucess, False otherwise.

pingSession(active=True)[source]¶

Pings an open session to tell the system that it’s still active. If the session hasn’t been pinged within 120 seconds, the system will close the session and you will have to open another one. It’s recommended that you ping every 30 seconds or so to keep the system from cleaning up your session. You can also let the system know whether the player is in an “active” or “idle” state within your game through this call. To do this, you pass an argument to the active variable. If it is set to True, then the player state will be set to active. If False, it will be set to idle. Return a boolean value: True if a session pinged with sucess, False otherwise.
